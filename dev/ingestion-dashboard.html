Decision read:

Process Frame

This page should be the primary place where new game materials are added, assessed, and turned into explicit build decisions. It should stay compact at the top level, let us click into detail by game, and keep showing what source families are still missing before design outruns evidence.

Artifact Collection Accession real source materials first: gameplay video, manuals, score tables, stills, cabinet art, and written reference pages.
  • preserved-source packages
  • checksums and manifests
  • source inventory updates
Analysis Turn sources into reviewable windows and evidence: contact sheets, waveforms, semantic notes, timing slices, and confidence gaps.
  • reference-lineage analysis folders
  • first-pass readmes
  • preview artifacts
Platinum Instantiation Plan Convert evidence into an explicit game plan that separates Platinum-owned host work from game-owned rules, timing, audio, and identity.
  • game-owned plan tracks
  • platform extension candidates
  • priority artifact hunts
Implementation Build only the smallest evidence-backed slice needed to prove the game on Platinum.
  • runtime slice
  • pack contracts
  • capture and review harnesses
Conformance Measure the game against its source lineage with runtime artifacts, personas, and visible review surfaces.
  • game-specific scorecards
  • runtime captures
  • hosted conformance views
Release Review Decide whether the game is ready for public claims, beta review, or continued private iteration.
  • release-lane posture
  • user-facing readiness read
  • next best moves
